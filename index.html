Network Schematics Tool
Instantly generate schematics for ICM catchment networks.
The Network Schematics tool streamlines the generation of schematics for any ICM catchment network, simplifying a process that is often time-consuming and prone to errors in large and complex networks. This tool empowers users to upload data files generated by running a Ruby script on the ICM network, allowing for extensive customization of the resulting schematics.

Users can enhance their schematics through a variety of intuitive actions, including Adding new elements. Deleting unwanted components, modifying or creating connections (edges) between nodes with click-and-drag functionality.

The tool supports HTML markup within elements, enabling users to incorporate a wide range of content, such as images, videos, and other chart components, into their schematics.

Additionally, the tool allows for the styling of nodes and connections, providing options to edit their appearance for improved visualization. Legends can be easily added and managed, enhancing the clarity and organization of the schematics.

Once all modifications are complete, users can save their work for future reference or download the final schematics in JPEG format. This ensures flexibility and ease of use throughout the design process.

Time Series Data Visualization and Manipulation Tool
A browser-based tool for visualizing and manipulating time series data.
This JavaScript tool allows users to visualize and manipulate time series data, such as telemetry data (rainfall, depth, flows, pump states, etc.), which is essential for modelling. Often, data received from stakeholders may have gaps or be in an incorrect format, making it unsuitable for immediate use in modelling processes. This tool addresses these challenges by allowing users to load the data, visualize it, and identify gaps or inconsistencies.
  1. The tool offers several data manipulation features, including:
    • Filling missing data using methods like interpolation, previous value replacement, constants, or zeros.
    • Choosing the largest data gaps to be filled, ensuring large gaps remain unchanged if desired.
    • Altering data frequency, such as converting 5-minute intervals to 1-minute or vice versa.
  2. Users can compare the processed data with the original raw data and add color-coded levels to the plot for better analysis.
  3. The processed data can be exported as a CSV file, which includes both the raw and processed data along with processing parameters for record-keeping.
  4. Plots can be downloaded as JPEG images for reporting purposes.
  5. The tool is optimized for large datasets, tested with up to 2.1 million data points (e.g., 20 years of 5-minute gapped rainfall data).
This browser-based tool requires no installation and provides an intuitive interface for efficient time series data handling and visualization.
Flow Verification Tool
A combination of Ruby and JavaScript application for flow verification in UK Horton runoff model-based catchments.
The Flow Verification Tool is a combination of a Ruby script and a JavaScript application designed to aid in the flow verification of fixed, new UK and Horton runoff model-based catchments.

The Ruby script generates multiple scenarios in the network, corresponding to different flow-generating parameters. The modeller simulates these scenarios using flow survey rainfall and exports the results in a specific format. These results are then loaded into the browser-based application along with the recorded flow survey data and a contributing subcatchments table.

Within the tool, the modeller can adjust various parameters, such as runoff areas, ground infiltration profiles, and infiltration areas, to observe corresponding changes in flows visualized against the recorded flow survey data. Once the desired match with observed flows is achieved, the tool allows users to download the subcatchment parameters that will replicate these flows in the ICM model.

Initially developed to verify small village catchments with flow monitors close to treatment works, the tool is versatile enough to assist in verification for larger catchments with adjustments to input parameters. As a web-based solution, it requires no installation, making it easily accessible for users.

Exclusive Subcatchment Selector [Ruby]
Generates selection lists of exclusive subcatchments draining to specific points of interest within a network.

This Ruby script automates the creation of selection lists for exclusive subcatchments or networks draining to user-labeled points of interest, such as flow and depth monitoring points and ancillaries, within a network model.

Exclusive subcatchments refer to those that drain directly to a specific point of interest, excluding any areas draining to upstream points of interest.

Identifying these exclusive subcatchments is crucial for projects like Verification, Optioneering, and survey scoping. Manual identification using built-in ICM tools, like US Tracing, can be time-consuming, challenging, and often prone to errors, especially in large and complex networks with multiple points of interest.

This script provides a seamless solution, generating accurate selection lists for exclusive subcatchments in seconds, regardless of the network's size or complexity. By significantly reducing the time and effort involved, it ensures perfect accuracy.

The python whatsapp messaging library created as part of this project uses the power of python selenium library to automates sending of WhatsApp messages to a or a list of WhatsApp registered phone numbers.

Capabilities:
1. Checks if the user (message sender) is logged into WhatsApp Web and prompts to login if not.
2. Notify timeouts and poor internet.
3. Send Messages to a list of phone numbers passed as phone_number:Message key:value pairs.
4. Make sure a message is sent within the prescribed time limit.
5. Returns a summary of the messages sent.

Input: A dictionary of phone_number_string : message_string pairs

Return value: A dictionary of summary of sending messages OR an Error_string

The library was written in 2021 and is now obsolete due to updates to WhatsApp Web.

Finite Element Analysis (FEA) of Fractures in Steel Structure (B. Tech Project)
Program to evaluate fractures in simple steel structures using finite element analysis in MATLAB.
The project attempts to study how fracture mechanics problems can be analyzed using the Finite Element Method by testing FEM results obtained by changing various finite element parameters like meshing style, element type, mesh density, etc. The project also tries to investigate the practical application of such an analysis.

Fracture Mechanics and Damage Tolerant Design

Fracture mechanics emerged as a result of experiences of unexpected failures of structures which were conventionally designed against failure by yield. The development of cracks due to inherent defects, geometric discontinuities resulting in stress concentrations, reduction in ductility due temperatures, fatigue due to cyclic loading, etc. eventually became critical and resulted in catastrophic failure. Conventional design approach assumes a defect free body and stresses, strains and displacements are found using the particular material deformation law. Effective stress (σ) is calculated as a function of the geometry (G), loading (L) and the material properties (M)-

$$\sigma = f(G, L, M) < \sigma_c$$

The Finite Element Method in Fracture Mechanics

The Finite Element Method (FEM) is extensively being used for numerically solving partial differential equations in various fields in engineering and science. Analytical solution of the fracture mechanics parameters changes drastically with change in the geometry and type of loading on the model. Arriving at the complex stress function requires partial guess work and partial experience, after which the remaining mathematics is so rigorous that it is impractical to do it for every analysis. The finite element method removes this difficulty, by acting as a semi-general-purpose solution method. Initial work done in analytical solutions to fracture mechanical problems by Tada, Paris and Irwin [1], Westergaard [2], and Williams [3] has allowed traditional finite elements to be modified, to suit the derived analytical solutions using techniques such as the quarter point shift, to simulate the 1/√r singularity towards the crack tip. Non-destructive testing methods may be used to discover and measure defects in real life structural components, following which a finite element simulation could be done, to predict the proximity to failure, and schedule consequent tests or even take decisions to repair or replace the component.
